# EAP Robot Settings — 8 格插槽貼入內容（正本）

> EAP 後台路徑：Service Management → Robot Settings → Prompt/Config Setting。
> 這 8 格不是 8 隻機器人，是同一條 RAG 管線的 8 個階段。
> **Generate Response 的正本是 [robot_setting.txt](robot_setting.txt)，本檔不重複**（單一真實來源規則）。
> 改完任何一格都要**新開 chat** 才生效（EAP_OPERATIONS.md §4.2）。

---

## 1. Contextual Combination（追問改寫）— 建議貼

官方用途：判斷新問題與對話紀錄的關聯，把追問改寫成完整問題。
我們的加值：改寫時**強制保留 ETF 代號與公告日期**，否則改寫後的問題會失去文件鎖定、觸發幻覺（§3.4 實測坑）。

```
請根據user的[問題]與對話紀錄。
1.如果[問題]與之前對話無關，輸出"NA"。
2.如果[問題]是延伸之前對話，請重寫成完整的問題。重寫時必須保留並補全：ETF 代號（如 0050、00878）與公告名稱、日期（如 112/10/31 收益分配公告）；禁止使用「它」「那檔」「這次」等代名詞。
例：上一輪在談 00878 收益分配公告，user 追問「那平準金占多少？」→ 重寫為「依據 00878 民國112/10/31 收益分配公告，收益平準金占比為多少？」
```

## 2. Scene Adjustment（優先搜尋指定欄位）— 保守貼

官方用途：把輸入改寫成「優先搜尋某類項目」。目前知識庫只有收益分配公告＋1 份期中報告，
所以只在配息相關問題時導流，其他輸出 NA（不改寫），避免誤導檢索。

```
若user的輸入與「配息、收益分配、平準金、除息、配息來源」相關，請改寫為：「請在收益分配公告文件中搜尋與[原輸入]相關的內容」。
其他情況一律輸出"NA"。
```

## 3. Semantic Adjustment（術語對照）— 建議貼

官方用途：把口語/俗稱換成**文件內使用的詞彙**（左→右）。公告用語是「收益分配」「收益平準金」、日期用民國紀年，這格直接提升檢索命中率。

```
如果出現以下詞彙，請替換為文件中使用的詞彙（左→右）；否則輸出"NA"。
配息 → 收益分配
除息 → 收益分配之除息
配息來源 → 收益分配組成（股利所得、利息所得、已實現資本利得、收益平準金）
平準金 → 收益平準金
殖利率 → 收益分配率
台積電 → 台積電（2330）
台灣50 → 元大台灣50（0050）
國泰永續高股息 → 國泰永續高股息（00878）
群益精選高息 → 群益台灣精選高息（00919）
科技優息 → 復華台灣科技優息（00929）
高息低波 → 元大台灣高息低波（00713）
西元年份 → 民國年份（西元−1911），例：2023年10月 → 112年10月
```

## 4. Scene Check 1（內容能否答題的 YES/NO 閘門）— 建議貼

官方用途：判斷檢索到的 chunk 能否回答問題，只輸出 YES/NO。
我們的加值：把代號↔名稱、民國↔西元、配息↔收益分配宣告為同義，避免明明檢索到了卻被判 NO 而落入拒答。

```
你是智能助手，這是是非題，只要回答"YES"/"NO"。
1.判斷[內容]是否可回答[問題項目]。
2.以下視為同義：ETF 代號與其基金名稱（如 00878＝國泰永續高股息）；民國與西元日期（112年＝2023年）；「配息」＝「收益分配」；「平準金」＝「收益平準金」。
3.[內容]與[問題項目]是同一問題的不同問法、或包含同義詞時，回答 YES。
4.只輸出判斷結果：YES/NO。
```

## 5. Setting for when unable to answer（查無資料時的罐頭訊息）— 已貼，建議升級

這格是**給使用者看的訊息**，不是 prompt。升級版順便教使用者正確問法（demo 防呆）：

```
目前知識庫（資料快照 2026-06-05）尚無此資料，無法回答。請改問 16 檔 ETF 的收益分配公告內容，並指明 ETF 代號與公告日期，例如：「依據 00878 民國112/10/31 收益分配公告，各類收益占比為何？」
```

## 8. Specific Context B（Industry Know-How）— 建議貼

```
- 配息來源四類：股利（股息）所得、利息所得、已實現資本利得、收益平準金；公告以占比（%）揭露，四類加總＝100%
```

## 貼完後驗收（每次改動都要跑）

1. **新開 chat**（舊 chat 用的是舊 setting 快照）。
2. 問「依據 00878 民國112/10/31 收益分配公告，各類占比？」→ 要命中 股利31.36／利息0.07／平準金5.71／資本利得62.86。
3. 追問「那 113Q1 那次呢？」→ 驗 Contextual Combination 有把代號/文件補回去。
4. 問「00878 配息的平準金占多少？」（口語版）→ 驗 Semantic Adjustment 換詞後仍能命中。
5. 問「我該買 0050 嗎？」→ 要被拒答條款攔截。
6. 問一個知識庫沒有的（如「00940 配息公告」）→ 要回 unable-to-answer 罐頭訊息，不是幻覺數字。
